## The best way to learn data engineering

We think the fastest way to actually *learn* data — SQL, modeling, warehouses, ETL, and
more — is **hands-on, on real, production-like data**, not toy tables you forget the
moment you close the tab.

Every lesson comes in more than one format, because people learn differently:

- 📖 **Read it** — a written walkthrough, for people who learn from text.
- 📺 **Watch it** — a short video of the same lesson, for people who prefer watching.
- 🧪 **Do it — the koans.** Fill-in-the-blank exercises you run yourself and watch go
  from red to **green**.

Read and watch are great for *understanding*. But the **koans are where it sticks** —
typing real queries against real data, predicting the answer, and proving you were right.
**If you genuinely want to learn, do the koans.** That's our whole philosophy, and it's
why this repo exists.

## Quickest way to get learning: DataPallas

Everything here is **free and open source**, and the fastest way to get a real data
playground *and* these lessons running on your machine is **DataPallas**:

👉 **https://datapallas.com/docs/learn-data-engineering**

It sets up a real database, sample data, and a place to write and run queries — so you
can go from zero to your first query in minutes.

## Do the koans (after DataPallas is set up)

```bash
cd C:/DataPallas/db/datazeus
koans.bat  sql S1 _00    # Windows   (course · series · episode — start here)
./koans.sh sql S1 _00    # macOS / Linux
```
Run all of a course with `koans.sh sql`, a series with `koans.sh sql 1`, or every
koan with `koans.sh`. Courses: `sql modeling etl warehousing dbt viz bi`.

Each koan shows you the value the query actually returned — fill in the blank with it,
run again, and watch it turn green. That's one lesson, learned by doing.
